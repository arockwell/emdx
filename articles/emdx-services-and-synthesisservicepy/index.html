    emdx: services and synthesis_service.py¶

    Overview¶

    emdx/services/synthesis_service.py provides shared AI-powered document synthesis capabilities for EMDX commands. It uses Claude's Opus model to intelligently merge multiple related documents into cohesive, well-organized articles while removing redundancy and preserving technical accuracy.

    The service powers two major KB management features: - emdx compact â€” Reduces KB redundancy by synthesizing duplicate/overlapping documents - emdx distill â€” Surfaces and synthesizes KB content for sharing with different audiences

    Key Concepts¶

    SynthesisService Class¶

    The core service class manages AI-powered document synthesis with these responsibilities:

    class SynthesisService:
    """Service for AI-powered document synthesis."""

    def synthesize_documents(
        self,
        doc_ids: List[int],
        purpose: str = "synthesis"
    ) -> SynthesisResult:
        """Synthesize multiple documents into one coherent document."""

    def estimate_cost(
        self,
        doc_ids: List[int]
    ) -> Tuple[int, float]:
        """Estimate tokens and cost before synthesis."""

    Design principles: - Always uses Claude Opus for quality-critical synthesis - Lazy-loads Anthropic client with import guards (optional [ai] dependency) - Truncates long documents to ~6000 characters to manage context window - Returns structured SynthesisResult with content, metadata, and usage statistics

    SynthesisResult Dataclass¶

    Tracks synthesis output and metadata:

    @dataclass
class SynthesisResult:
    success: bool
    new_doc_id: int | None
    source_doc_ids: List[int]
    title: str
    content: str  # Synthesized markdown content
    error: str | None = None
    tokens_used: int = 0
    cost_usd: float = 0.0
    model: str = "claude-opus-4-5-20250929"

    Document Archival: superseded Tag System¶

    When documents are synthesized via emdx compact, original source documents are archived, not deleted:

    1. Tagging: Sources receive superseded tag + superseded-by:ID tag linking to new doc
    2. Reversibility: Users can view/restore archived docs anytime
    3. Traceability: Full provenance chain via tag metadata

    Example:

    # Compact docs 42, 43, 44 â†’ creates doc 100
emdx compact 42 43 44

# Original docs now tagged:
# #42: superseded, superseded-by:100
# #43: superseded, superseded-by:100  
# #44: superseded, superseded-by:100

    This pattern enables KB cleanup while maintaining full audit trail.

    Architecture & Design Decisions¶

    Shared Service Pattern¶

    SynthesisService is designed as reusable infrastructure shared across multiple commands:

    Command Use Case Key Difference
    compact Reduce redundancy Merges similar/duplicate documents
    distill Share knowledge Synthesizes for specific audiences (me/docs/team)

    Both commands use the same synthesis engine but with different: - Prompts â€” compact focuses on deduplication; distill on audience-appropriate formatting - Selection â€” compact uses TF-IDF clustering; distill uses search/tags - Output â€” compact archives originals; distill creates standalone briefs

    Cost Management Strategy¶

    AI synthesis is quality-critical but expensive, so the service implements multiple cost controls:

    1. Always Opus â€” Uses highest-quality model (claude-opus-4-5-20250929) because synthesis errors are costly
    2. Token estimation â€” estimate_cost() calculates approximate cost before API call
    3. Transparent billing â€” Returns actual token usage and USD cost in SynthesisResult
    4. Free discovery â€” emdx compact --dry-run uses zero API calls (local TF-IDF only)

    Pricing (approximate): $15/$75 per million tokens (input/output)

    Prompt Engineering¶

    The synthesis prompt instructs Claude to:

    1. Identify core topic/theme across all documents
    2. Extract and preserve best insights from each
    3. Remove redundant information (keep best version)
    4. Organize content logically
    5. Use clear, consistent writing style

    Critical directive: "Do NOT add meta-commentary like 'This document synthesizes...' â€” just output the content."

    This produces clean, production-ready markdown suitable for direct use.

    Import Guards for Optional Dependencies¶

    Since anthropic SDK is an optional dependency (part of [ai] extra), the service uses defensive imports:

    try:
    import anthropic
    HAS_ANTHROPIC = True
except ImportError:
    HAS_ANTHROPIC = False

def _require_anthropic() -> None:
    if not HAS_ANTHROPIC:
        raise ImportError(
            "anthropic is required for synthesis features. "
            "Install it with: pip install 'emdx[ai]'"
        )

    This keeps core EMDX lightweight while providing helpful errors when AI features are used without dependencies.

    Document Loading and Preparation¶

    def _fetch_documents(self, doc_ids: List[int]) -> List[DocumentContent]:
    """Load documents by ID with their tags."""
    docs = []
    for doc_id in doc_ids:
        doc = get_document(doc_id)
        if doc:
            tags = get_document_tags(doc_id)
            docs.append(DocumentContent(
                doc_id=doc_id,
                title=doc.get("title", f"Document #{doc_id}"),
                content=doc.get("content", "")[:6000],  # Truncate
                tags=tags
            ))
    return docs

    Key details: - Fetches full document content from DB - Retrieves associated tags for context - Truncates content to 6000 chars to manage context window - Does not update access tracking (unlike get_document() normally)

    Tag Management¶

    When archiving source documents:

    def archive_originals(
    self,
    source_doc_ids: List[int],
    new_doc_id: int
) -> int:
    """Archive source documents after synthesis."""
    for doc_id in source_doc_ids:
        add_tags_to_document(doc_id, [
            'superseded',
            f'superseded-by:{new_doc_id}'
        ])

    Tag filtering rules: - Status tags (active, done, blocked) are NOT transferred to synthesized doc - superseded tag is added to originals - synthesized tag is added to new doc - superseded-by:ID creates traceable link

    Audience-Aware Synthesis (distill command)¶

    The distill command extends synthesis with audience-specific prompts:

    class Audience(Enum):
    ME = "me"           # Dense, technical, includes doc IDs
    DOCS = "docs"       # Formal, standalone documentation
    COWORKERS = "coworkers"  # Team briefing format

    Each audience mode uses different system prompts: - ME: Bullet points, informal tone, includes source doc IDs for drill-back - DOCS: Technical documentation with clear structure and examples - COWORKERS: Executive summary and action items

    emdx compact Command¶

    Uses SynthesisService to reduce KB redundancy through intelligent document merging.

    Architecture:

    Discovery Layer (free, local TF-IDF) 
    â†’ Synthesis Layer (Claude API)
    â†’ Archive Layer (superseded tags)

    Usage:

    emdx compact --dry-run              # Free discovery only
emdx compact 42 43 44               # Synthesize specific docs
emdx compact --auto                 # Process all clusters
emdx compact --topic "auth" --auto  # Topic-filtered synthesis

    emdx distill Command¶

    Uses SynthesisService to create audience-appropriate briefs from KB content.

    Usage:

    emdx distill 'authentication'              # Topic search
emdx distill --tags 'security,active'      # Tag filtering
emdx distill --for docs 'API design'       # Docs audience
emdx distill --for coworkers 'status'      # Team briefing
emdx distill 'auth' --save --title "Brief" # Save to KB

    superseded-by:id Tag Pattern¶

    The superseded-by:ID tag pattern creates a traceable provenance chain:

    1. When doc 100 synthesizes docs 42, 43, 44
    2. Each source gets tagged: superseded, superseded-by:100
    3. Users can find synthesis source via tag search: emdx find --tags superseded-by:100
    4. Reversible: remove tags to "unarchive" originals

    This pattern balances KB cleanup with full audit trails.
