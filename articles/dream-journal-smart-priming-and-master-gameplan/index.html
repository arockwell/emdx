    Dream Journal: Smart Priming and Master Gameplan¶

    Overview¶

    EMDX is fundamentally Claude's memory system â€” a self-organizing knowledge base where AI agents populate and humans curate. The Dream Journal initiative transforms emdx from passive document storage into an active intelligence system that learns, organizes, and surfaces information automatically. Three core features work together: overnight consolidation (Dream Journal), context-aware session initialization (Smart Priming), and duplicate prevention (Contextual Save).

    Design Philosophy: Claude-First

    Every decision optimizes for machine-readability, minimal token cost, composability, and cross-session intelligence. The hot path is Claude Code â†’ emdx CLI â†’ stdout â†’ Claude's context window. Features prioritize:

    • Non-blocking UX: Warnings don't prevent saves
    • Token efficiency: Chunk-level retrieval, concise output
    • Autonomous curation: Systems run without human oversight
    • Discoverability: Context surfaces proactively

    Key Concepts¶

    The Knowledge Management Problem¶

    Claude sessions operate in isolation, leading to: - Duplicate research: 60-80% content overlap across documents - Cold starts: Every session begins without awareness of prior work - Knowledge fragmentation: Insights scattered, connections invisible - Knowledge rot: Important decisions forgotten, outdated docs persist

    The Dream Journal initiative addresses all four through automated consolidation, relationship discovery, and intelligent retrieval.

    The Three-Tier Intelligence Model¶

    Tier 1: Save-Time Intelligence - Contextual Save: Prevents duplicates by checking similarity on save - Auto-Linking: Creates bidirectional links between related docs - Result: KB self-organizes as it grows

    Tier 2: Search Intelligence - Hybrid Search: Combines keyword + semantic, returns chunks not whole docs - RAG Q&A: Synthesizes answers from multiple documents via emdx ai ask - Result: Finding information becomes conversational

    Tier 3: Maintenance Intelligence - Smart Priming: Context-aware session startup - Dream Journal: Overnight consolidation and pattern detection - Knowledge Decay: Surfaces stale but important documents - Result: KB maintains itself

    Hybrid Search Algorithm¶

    Default mode combines three signals:

    # emdx/services/hybrid_search.py
def compute_hybrid_score(keyword_score, semantic_score, in_both):
    base = 0.4 * keyword_score + 0.6 * semantic_score
    if in_both:
        base += 0.15  # Overlap boost when term appears in both modes
    return base

    Score Normalization: - FTS5 scores: rank / max_rank (0-1 range) - Semantic scores: Cosine similarity already 0-1 - Configurable alpha weights in constants.py

    Mode Detection: - Embeddings exist â†’ Hybrid (default) - No embeddings â†’ Keyword (FTS5 fallback) - Explicit --mode semantic|keyword|hybrid

    CLI Usage:

    $ emdx find "authentication patterns" --extract

    Auto-Linking Logic:

    # emdx/services/linking_service.py
def create_links(doc_id: int) -> List[Link]:
    # Find top 5 similar docs (>50% threshold)
    similar = hybrid_search(get_doc(doc_id).content, limit=5)

    # Create bidirectional links
    for match in similar:
        if match.score > 0.5:
            create_link(doc_id, match.id, score=match.score)
            create_link(match.id, doc_id, score=match.score)

    Prevents over-connection: Max 5 links per document, minimum 50% similarity.

    Contextual Save Service¶

    Prevents duplicate research through pre-save similarity checks:

    # emdx/services/contextual_save.py
@dataclass
class CheckResult:
    similar_docs: List[SimilarDoc]  # Top N matches
    recommendation: str              # "Update #6085" or "Save as new"
    suggested_tags: List[str]        # From neighbor tags
    suggested_project: str | None    # Inferred from content

def check_before_save(title: str, content: str) -> CheckResult:
    similar = hybrid_search(f"{title} {content}", limit=10)
    duplicates = [d for d in similar if d.score > 0.8]

    if duplicates:
        return CheckResult(
            similar_docs=duplicates,
            recommendation=f"Duplicate of #{duplicates[0].id}. Update instead?",
            suggested_tags=extract_common_tags(duplicates[:3]),
            suggested_project=infer_project(content)
        )

    Smart Priming Service¶

    Context-aware session initialization using clustering and recency:

    # emdx/services/priming_service.py
class PrimingService:
    def get_context(self, lookback_days=7, n_clusters=5, max_docs=10):
        # 1. Get recently accessed docs
        recent = get_recent_docs(lookback_days)

        # 2. Cluster by topic using MiniBatchKMeans
        embeddings = [doc.embedding for doc in recent]

        # 3. Rank tasks by cluster coherence
        tasks = get_ready_tasks()

        # 4. Return grouped context
        return PrimingContext(
            tasks_by_cluster=group_by(tasks, 'cluster_id'),
            recent_docs=recent[:max_docs],
            knowledge_map=build_tag_coverage(recent)
        )

    Knowledge Decay Detection¶

    Surfaces forgotten but important documents:

    # emdx/services/decay_service.py
def compute_staleness(doc: Document) -> float:
    days_since_access = (now() - doc.accessed_at).days
    age_weight = 0.4 * (days_since_access / 365)

    link_count = count_incoming_links(doc.id)
    link_weight = 0.3 * (1 - link_count / max_links_seen)

    recent_activity = count_related_activity(doc.id, days=30)
    activity_weight = 0.3 * (1 - recent_activity / max_activity_seen)

    return min(1.0, age_weight + link_weight + activity_weight)

    Dream Journal: Overnight Consolidation¶

    Core Functions: 1. Duplicate detection & merging (>80% similarity) 2. Tag normalization ("auth", "authentication" â†’ "authentication") 3. Automatic relationship discovery via clustering 4. Staleness detection and flagging 5. Summary generation from document clusters

    Planned Implementation:

    # Cron job (runs at 2am)
0 2 * * * emdx maintain consolidate --auto

    Consolidation Pipeline:

    # emdx/commands/maintain.py
def run_consolidation():
    # 1. Find duplicate clusters
    duplicates = find_similar_clusters(threshold=0.8)

    # 2. Merge each cluster
    for cluster in duplicates:
        merged = merge_documents(cluster, strategy="full_merge")
        archive_sources(cluster.doc_ids)
        create_consolidation_record(cluster.doc_ids, merged.id)

    # 3. Normalize tags
    tag_mapping = detect_tag_variants()
    standardize_tags(tag_mapping)

    # 4. Rebuild indexes
    rebuild_embeddings()
    rebuild_links()

    Database Schema:

    CREATE TABLE consolidation_history (
    id INTEGER PRIMARY KEY,
    source_doc_ids TEXT,          -- JSON array of merged doc IDs
    result_doc_id INTEGER,
    similarity_score REAL,
    merge_strategy TEXT,          -- "full_merge", "summary", "link_only"

    Merge Strategies: - full_merge: Combine content, preserve all details - summary: LLM-generated overview linking to sources - link_only: Keep separate, add "superseded by" links

    Safety: Never auto-delete. Sources archived (soft delete) with full provenance tracking.

    Lazy Loading Pattern¶

    Heavy dependencies (sentence-transformers, scikit-learn) use TYPE_CHECKING guards to avoid startup penalties:

    from __future__ import annotations
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from emdx.services.embedding_service import EmbeddingService

class HybridSearchService:
    def __init__(self):
        self._embedding: EmbeddingService | None = None

    def _get_embedding(self) -> EmbeddingService:
        if self._embedding is None:
            from emdx.services.embedding_service import EmbeddingService
            self._embedding = EmbeddingService(self.db_path)
        return self._embedding

    Rationale: Avoid importing torch/transformers for simple CLI commands like emdx view.
