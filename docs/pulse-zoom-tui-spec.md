# Pulse+Zoom TUI Design Specification

## Executive Summary

The Pulse+Zoom TUI is a three-level zoom interface for task and workflow management in EMDX. It provides semantic zooming from high-level overview (Zoom 0) to focused task work (Zoom 1) to deep log analysis (Zoom 2), with smooth state preservation across transitions.

**Design Philosophy:**
- **Semantic Zoom**: Each level reveals more detail about fewer things
- **State Preservation**: Never lose context when zooming
- **Keyboard-First**: Every action accessible via memorable shortcuts
- **Information Density**: Maximize useful information per screen character

## 5. Keyboard Shortcut System

### 5.1 Design Principles

1. **Vim-like navigation** - j/k/h/l for movement
2. **Mnemonic keys** - First letter of action (r=run, e=edit, d=delete)

### 8.3 Fast Log Output (100+ lines/sec)

**Problem:** UI can't keep up with rapid log output.

**Solutions:**

```
LOG STREAMING OPTIMIZATION
==========================

1. BATCHING
   - Collect lines for 100ms before rendering
   - Render batch as single update
   - Max 50 lines per batch (discard oldest if overflow)

2. THROTTLING
   - In LIVE mode: Max 20 updates/second
   - In non-LIVE mode: No streaming, manual refresh

3. ACTIVITY INDICATOR
   Instead of showing all lines, show activity sparkline:

   â–â–â–â–Œâ–Œâ–Œâ–â–â–â–Œâ–Œâ–‘â–‘â–‘â–â–â–â–Œâ–Œâ–Œâ–Œâ–Œ (100+ l/s burst)

   User can scroll up to see buffered content

4. BUFFER MANAGEMENT
   - Keep last 10,000 lines in memory
   - Older lines: "... 5,432 earlier lines (scroll up to load)"
   - Lazy load from file when scrolling up

5. FILTER IN STREAM
   - Apply filters during streaming, not after
   - Regex filters compiled once, reused
   - Skip lines that don't match filter

6. RENDERING OPTIMIZATION
   - Use RichLog with markup=False for speed
   - Batch syntax highlighting (every 500ms)
   - Virtualize: only render visible lines + 100 buffer

Performance Targets:
  - 100 lines/sec: Smooth, all lines visible
  - 1000 lines/sec: Batched, activity indicator, smooth scroll
  - 10000+ lines/sec: Sampling mode, show every Nth line
```
